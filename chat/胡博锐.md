## 项目总结：ARY (Agent Racing Yard)

### 📌 项目定位

**ARY**（Agent Racing Yard）是一个**智能体骑行赛事平台**，面向 Agentic Development 时代。它的核心使命是：

> "通过一场标杆赛事，把 Agent Riding Skill 变成可观察、可展示、可评审、可证明的能力资产。"

传统 Hackathon 只记录"报名 → 提交作品 → 评审 → 排名"，而 ARY 额外记录：
- 谁在骑 Agent、如何骑、骑得是否有效
- 成本、进度、风险如何
- 如何纠偏、最终交付了什么

---

### 🏗️ MVP 五大交付承载

| 承载 | 说明 |
| --- | --- |
| **Public Site 公开端** | 首页/Race Gallery、赛事详情页、实况大厅(Live Hall)、作品列表、赛果、评审总结、骑手档案、合作入口 |
| **Race Console 赛事控制台** | Organizer(办赛)、Rider(参赛)、Judge(评审) 三个视图 |
| **Admin Console 账号控制台** | 用户管理、角色(`User.roles`)维护 |
| **Screen Console 大屏控制台** | 现场/课堂/直播大屏展示控制 |
| **Riding Intelligence 底层能力** | CA 数据接入、成本/进度/风险指标、Evidence、Projection、报告生成 |

---

### 👥 核心用户角色

- **Public Visitor** — 公众访客，浏览赛事和传播
- **Rider** — 参赛骑手，使用 CA 完成比赛
- **Organizer** — 主办方，创建和管理赛事
- **Judge** — 评委，评审作品和骑行能力
- **Admin** — 管理员，维护用户角色
- **Screen Operator / Data Maintainer** — 操作职责（不新增独立 role）

---

### ⚙️ 关键业务规则

1. **Registration approved → 系统自动幂等生成 RaceProject**
2. **CA 接入不是参赛资格硬门禁** — 失败/未配置只进入评审前风险提示，不阻断提交和评奖
3. **CAConnection 可在参赛过程中动态新增** — 必须先登记和握手
4. **Projection 不是最终事实源** — 可重算，只服务过程展示
5. **原始 CA Session 默认不公开**
6. **Gallery-first** — 首页以赛事资产为主体，不是后台索引
